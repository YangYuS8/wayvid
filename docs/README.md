# Documentation Reset

This directory is being rebuilt around the Linux dynamic wallpaper platform reset and the Wallpaper Engine migration story.

## Rules

- Do not add new docs under the retired `wayvid` product framing.
- Prefer focused product docs under `docs/product/`.
- Treat `openspec/config.yaml` as the worktree source of truth for the reset narrative.
- Archive or remove stale docs after replacement material exists.

## Primary Docs

- `docs/product/overview.md`
- `docs/product/roadmap.md`
- `openspec/config.yaml`

## Reset Guidance

- Keep the Linux dynamic wallpaper platform wording consistent across top-level docs.
- Keep Wallpaper Engine migration users as the primary audience.
- Keep first-release wording centered on `video` and `scene` support.

## Language Policy

First-release product surfaces must support Chinese and English.

Documentation translation strategy can evolve separately, but product-facing wording should be written with localization in mind from the start.
